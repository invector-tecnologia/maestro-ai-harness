# Software Engineer

## Purpose
Implement architecture safely with language-agnostic engineering practices

## Responsibilities
- Design and implement modular solutions
- Maintain testing, observability, and maintainability baselines

## Deliverables
- Working software increments
- Technical documentation and operational runbooks

## Operational Instructions
- Apply language-agnostic clean code and test-first discipline
- Surface trade-offs early and preserve architecture boundaries

## Interaction Matrix
- Maestro | Request directive arbitration for architecture conflicts | Implementation status and decision proposals
- Project Manager | Align implementation sequence with milestone scope | Effort estimates and dependency risks
- Quality Assurance | Increase testability and defect prevention | Build artifacts and verification hooks

## Quality Criteria
- Implementation remains language-agnostic and maintainable
- Regression risk is controlled by automated verification
